VORTEX

Privacy Policy — Last updated March 31, 2026

Summary

Vortex does not collect, store, or transmit any personal data. Period.

Audio Processing

All audio analysis happens locally on your device using the Web Audio API and Android AudioPlaybackCapture. Your music files never leave your device. No audio is recorded, uploaded, or shared.

Data Collection

Vortex does not use analytics, tracking pixels, advertising SDKs, or any form of telemetry. There are no user accounts, no sign-in, and no server-side components beyond static file hosting.

Network Requests

The app makes no network requests during normal use. The only network activity is loading the app itself from the hosting provider (Netlify for the web version). The Android app runs entirely offline after installation.

Permissions (Android)

RECORD_AUDIO — Required by Android for the system audio capture feature (AudioPlaybackCapture). This permission is only used when you explicitly activate live audio capture. No microphone data is stored or transmitted.

READ_MEDIA_AUDIO / READ_EXTERNAL_STORAGE — Used to load audio files from your device for playback and visualization. Files are accessed read-only and never modified.

FOREGROUND_SERVICE — Required to keep the audio capture service running while visualizing system audio.

Third-Party Services

None. Vortex includes no third-party SDKs, libraries, or services that collect data.

Open Source

Vortex is open source. You can verify all of the above by reading the code: github.com/QAv2/vortex-web
